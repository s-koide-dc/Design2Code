# syntactic_analyzer 設計ドキュメント (Class: SyntacticAnalyzer)

## 1. 目的 (Purpose)

本モジュールは、`morph_analyzer` によって生成されたトークンリストを受け取り、日本語の構文構造（係り受け構造）を解析する。文を「文節」にまとめ、文節間の依存関係を特定することで、自然言語の指示文を「論理的な木構造」として表現し、後続の `CodeSynthesizer` が正確な論理式やデータフローを構築できるようにする。

## 2. 構造化仕様 (Structured Specification)

### 2.1. 入力 (Input)
- `analysis.tokens` を含むコンテキストオブジェクト。

### 2.2. 出力 (Output)
- `analysis.syntax_tree` フィールドが更新されたコンテキスト。
- `syntax_tree` は以下の構造を持つ文節（Node）のリストである。
    - `id`: 文節のインデックス (0, 1, 2...)
    - `surface`: 文節の表面文字列（例：「価格が」）
    - `tokens`: 含まれる形態素トークンのリスト
    - `head`: 係り先の文節 ID。文末（Root）の場合は -1
    - `dep_type`: 依存関係の型（SUBJ: 主語, OBJ: 目的語, MOD: 修飾, COMP: 比較, ROOT: 根）

### 2.3. 中心的なロジック (Core Logic)

1.  **文節分割 (Chunking)**:
    - 形態素トークンを走査し、「自立語 ＋ 付属語（助詞・助動詞）」の単位で文節を作成する。
    - 接頭辞や記号の結合も考慮する。
2.  **係り受け解析 (Dependency Parsing)**:
    - 各文節の末尾にある助詞を基に、係り先を決定する（決定論的ルール）。
    - **基本ルール**:
        - 原則として、自分より後ろにある最も近い「述語（動詞・形容詞）」に係る。
        - **「の」**: 直後の名詞句に係る。
        - **「が」「を」「に」「へ」「と」**: 文末方向の動詞に係る。
        - **「より」**: 比較表現（大きい、超える等）に係る。
        - **「かつ」「または」**: 論理的な結合（AND/OR）としてマークする。
3.  **木構造の構築**:
    - 全ての文節の `head` を確定させ、単一の根を持つ木（または森）を形成する。

### 2.4. テストケース (Test Cases)

-   **ハッピーパス**:
    -   **入力**: 「価格が 100 より 大きい ユーザーを 抽出する」
    -   **期待される構造**:
        - `抽出する` (Root)
            - ── `ユーザーを` (OBJ)
                - ── `大きい` (MOD)
                    - ── `価格が` (SUBJ)
                    - ── `100 より` (COMP)
