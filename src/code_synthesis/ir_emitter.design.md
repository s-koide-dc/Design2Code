# IR Emitter Design Document

## 1. Purpose
The `IREmitter` converts the structured Intermediate Representation (IR) tree generated by `IRGenerator` into concrete C# code. It employs a recursive emission strategy that preserves beam search candidates across block boundaries (loops, conditions, wrappers), ensuring a globally optimal code structure while maintaining local context awareness.

## 2. Structured Specification

### Input
- `ir_tree` (Dict): The IR tree containing logical nodes (`ACTION`, `LOOP`, `CONDITION`, `WRAPPER`).
- `initial_candidates` (List[Dict]): Initial synthesis paths (typically one root path).
- `beam_width` (int): The number of top candidates to preserve at each step.

### Output
- `candidates` (List[Dict]): A list of updated synthesis paths, each containing the generated code lines, variable scope state, and accumulated score.

### Core Logic
1.  **Recursive Traversal (`_emit_nodes`)**:
    -   Iterates through the list of IR nodes.
    -   Dispatches to specific handlers (`_emit_loop`, `_emit_condition`, `_emit_wrapper`) or delegates to `CodeSynthesizer` for `ACTION` nodes.
    -   Prunes candidates to `beam_width` after each node processing to manage combinatorial explosion.

2.  **State Preservation**:
    -   Crucially, `IREmitter` modifies the `path` dictionary in place (or copies it) to track:
        -   `combined_lines`: The generated C# code.
        -   `type_to_vars`: Available variables by type.
        -   `block_stack` & `block_vars`: Scoped variable management.
        -   `in_loop` (bool): Flag indicating if processing is currently within a loop body (used for context-aware scoring).
        -   `current_intent` (str): The intent derived from the current IR node (e.g., `LOOP_BODY`).

3.  **Block Handling**:
    -   **Loop (`_emit_loop`)**:
        -   Identifies the best source collection using `CodeSynthesizer`'s variable tracking.
        -   **Enhanced Entity Inference**: If the `target_entity` is generic or missing, it infers the element type from the source collection's type (e.g., extracting `Order` from `IEnumerable<Order>`).
        -   Sets `in_loop = True` and pushes a new scope.
        -   Recursively calls `_emit_nodes` for the loop body.
        -   Pops the scope and appends the closing brace `}`.
    -   **Condition (`_emit_condition`)**:
        -   **Contextual Entity Resolution**: Infers the `target_entity` from the active loop context if necessary.
        -   **Schema Fallback**: If no local POCO properties are defined, it falls back to the global `entity_schema` for property mapping.
        -   Generates the logic expression for `if`.
        -   Handles both `body` (if) and `else_body` blocks recursively.
    -   **Wrapper (`_emit_wrapper`)**:
        -   Wraps the body in a resilience block (e.g., `Polly.Policy...`).
        -   Adds necessary `using` statements and dependencies.

4.  **Intent & Context Propagation**:
    -   Injects `current_intent` into the path before processing actions.
    -   **Specific Intent Preservation**: Ensures that if a node has a specific intent (like `DISPLAY`), it is preserved even inside loops, rather than being blindly overwritten by `LOOP_BODY`.
    -   Enriches `action_name` with `main_poco` when inside a loop to favor item-level operations during method search.

### Test Cases
-   **Nested Loop**: Verify that inner loop actions correctly identify variables from the outer loop scope.
-   **Conditional Logic**: Ensure `if/else` blocks are correctly opened/closed and that variable scoping rules (hoisting) are respected.
-   **Wrapper Generation**: specific checking of `Polly` syntax generation.

## 3. Dependencies
-   `CodeSynthesizer`: Shared logic for variable management (`_copy_path`, `_exit_scope`) and action processing (`_process_action_node`).
-   `TypeSystem`: For type compatibility checks during source selection.
