# IR Patcher Design Document

## 1. Purpose
The `IRPatcher` [Phase 23.4] is a mechanism for self-correction within the synthesis loop. It accepts high-level "hints" generated by the `Replanner` (based on error analysis or reasoning) and applies targeted modifications to the Intermediate Representation (IR) tree. This allows the system to fix logic gaps, resolve missing dependencies, or correct intent misclassifications without restarting the entire synthesis process.

## 2. Structured Specification

### 2.1 Inputs
- **ir_tree** (`Dict[str, Any]`): The current IR tree structure (logic nodes, data sources, etc.).
- **hints** (`List[Dict[str, Any]]`): A list of patch instructions. Each hint contains a `patch` dictionary with a `type` and specific parameters.

### 2.2 Output
- **Patched IR** (`Dict[str, Any]`): The modified IR tree.

### 2.3 Core Logic

#### 2.3.1 Patch Dispatch (`apply_patches`)
Iterates through `hints` and executes logic based on `patch.type`:
1.  **FORCE_INTENT_RESOLUTION**: Calls `_patch_intent` to override a node's intent (e.g., GENERAL -> PERSIST).
2.  **REBIND_INPUT_LINK**: Calls `_patch_links` to explicitly connect data flow between nodes.
3.  **ADD_POCO_PROPERTY**: Calls `_patch_entities` (Stub) to extend data models.
4.  **ENSURE_FIELD_OR_LOCAL**: Calls `_patch_required_fields` to inject missing dependency fields.
5.  **FORCE_VARIABLE_BINDING**: Updates node semantics to prioritize variable usage over literals.
6.  **FIX_LOGIC_GAPS**: Marks nodes for retry/logic re-evaluation.

#### 2.3.2 Node Traversal (`_find_node`)
-   Recursively searches the `logic_tree` (including `children` and `else_children`) to find a node by its `id`.

#### 2.3.3 Field Injection (`_patch_required_fields`)
-   Maps missing symbols (e.g., `HttpClient`) to standard field names (e.g., `_httpClient`).
-   Appends the field name to `ir["required_fields"]` if not present.

#### 2.3.4 Intent Correction (`_patch_intent`)
-   Locates target node.
-   If method name implies storage (Save/Update/Delete), forces `intent="PERSIST"`.
-   Otherwise, defaults to "ACTION".

### 2.4 Test Cases

#### Happy Path
1.  **Fix Missing Field**:
    -   Hint: `type="ENSURE_FIELD_OR_LOCAL"`, `name="HttpClient"`.
    -   Result: `ir["required_fields"]` contains `_httpClient`.
2.  **Force Intent**:
    -   Hint: `type="FORCE_INTENT_RESOLUTION"`, `target_id="node1"`, `method="Save"`.
    -   Result: Node "node1" has `intent="PERSIST"`.
3.  **Link Nodes**:
    -   Hint: `type="REBIND_INPUT_LINK"`, `target_id="n2"`, `source_id="n1"`.
    -   Result: Node "n2" has `input_refs=["n1"]`.

#### Edge Cases
1.  **Node Not Found**:
    -   Hint targets non-existent ID.
    -   Result: Patch is silently ignored (safe failure).
2.  **Unknown Patch Type**:
    -   Hint has `type="UNKNOWN"`.
    -   Result: Ignored.
