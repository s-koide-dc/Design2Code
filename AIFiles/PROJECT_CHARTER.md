# Project Charter: ローカル対話応答AI

## 1. ビジョン (Vision)

古典的なパイプラインアーキテクチャに基づき、ユーザー個人の環境で軽快に動作する、パーソナルな対話応答AIを創造する。

## 2. ミッション (Mission)

*   特定のハードウェアやクラウドサービスに大きく依存せず、標準的なPCスペックで実行可能な対話応答AIを構築する。
*   「形態素解析」「構文解析」「意味理解」「応答生成」といった独立したモジュールの連鎖（パイプライン）により、ユーザーの入力意図を解釈し、応答を組み立てるだけでなく、**特定のアクション（ファイルの読み書き、コマンド実行等）の実行**を目指す。
*   設計書（自然言語）からの**コード/プロジェクト生成**を中核機能として位置づけ、決定論的な生成を前提とする。
*   語彙や知識の基盤として、JMDictやオープンなコーパスなどのフリー資産を積極的に活用し、拡張性と透明性を確保する。

## 3. 成功基準 (Success Criteria)

*   ユーザーが入力した日本語の文章を、形態素解析モジュールが単語単位に正しく分割できる。
*   簡単な質問に対し、パイプライン全体を通して文脈的に意味のある応答文を生成できる。
*   パイプラインの各処理モジュールが、単体でテスト可能である。
*   辞書データを追加・更新することで、AIの語彙を拡張できる。

## 4. 将来のビジョン: 高性能・軽量エージェントへの進化 (Evolution to Lightweight Agent)

本プロジェクトは、低負荷・高速応答という特性を維持したまま、ユーザーの作業を支援する「エージェント」への進化を目指す。

*   **意味理解の深化**: 重厚なLLMに頼らず、軽量な文章ベクトルモデル（BERTのサブセット等）や独自のルールエンジンにより、文脈の理解精度を極限まで高める。
*   **アクション・パイプライン**: 対話から正確にユーザーの意図（ファイル書き換え、タスク実行等）を読み取り、安全かつ確実にコード操作やシステム操作へ繋げる。
*   **決定論的な推論**: 確率的な生成ではなく、ルールと知識ベースに基づいた「正確で予測可能な」推論ロジックを拡充する。
*   **設計書駆動の生成**: 自然言語の設計書を入力とし、同一入力から同一出力が得られる生成パイプラインを維持する。
